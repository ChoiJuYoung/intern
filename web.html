1. how to use css

우선은 HTML에서 어떻게 css를 사용하느냐를 먼저 보겠습니다.

INLINE STYLE SHEET: <p style="color: blue">Like this</p>처럼 태그에 삽입


LINKING STYLE SHEET: <head>에 <link href="style.css" type="text/css" rel="stylesheet">로 CSS파일 연결


INTERNAL STYLE SHEET: <head>에 <style>과 </style>사이에 넣기


식으로 사용이 가능합니다.


2. 어떤 형식으로 쓰나요?

기본적인 구조는 다음과 같습니다.

 p {
   font-family: serif;
 }


즉, 어떤 것에 적용될 지를 미리 적어둔 뒤, 어떤 내용을 적용할 지 적어주게 됩니다.
ID는 #, class는 .과 함께 사용합니다.
가령, id가 test라면,

 #test { font-family: serif; }


식으로 적어주며, class가 container라면,

 .container { font-family: serif; }


와 같이 사용해주게 됩니다. 태그는 단독으로 적어주면 됩니다.
다만, 여러가지가 중첩돼서 들어가는 경우에는 문제가 생길 수 있습니다.
이에 대해서는, <p id="test" class="hello">라는 친구가 있다면
가장 우선해서 적용되는 것이 태그인지 id인지 class인지는 정해져 있습니다.
이 때의 specificity는 ID > class > tag순입니다.
또한, nested된 태그는 non-nested보다 우선됩니다.
nested란, 다음의 코드를 통해 볼 수 있습니다.

 <div id="test">
    <p class="container">
      CONTENT
    </p>
 </div>


위의 코드에서, CONTENT는 div태그 속의 p태그에 있습니다.
이 때, p태그를 nested되었다 라고 합니다.
전체 p가 아닌 위와 같은 경우만을 특별하게 취급해주기 위해서는

 div p.container {
   font-family: serif;
 }


과 같이 기재해주면 됩니다.
여기서 주의할 점은, p와 .container가 떨어져 있는 것이 아니라, p.container로 붙어있다는 점입니다.
만약, p와 .container가 떨어져 있다면, 위의 코드는
우리가 의도했던 div태그 속의 container클래스인 p태그
가 아닌, div태그 속의 p태그 속의 container클래스
라는 의미가 되어버립니다.



3. Font에 관하여

다음으로는 폰트에 관하여 보도록 하겠습니다.
가장 먼저 볼 수 있는 친구는 font-family입니다.
이 친구는, 폰트를 설정합니다.

font-family: serif;


와 같이 입력 시, serif 폰트 중 임의의 폰트가 설정되게 됩니다.

serif와 sans serif에 관해서는 위의 이미지를 참조하시면 됩니다.
요약하면,
serif는 각 글자의 끝마다 추가적인 디테일이 들어갑니다.
Times New Roman이나 Georgia가 이러한 부류입니다.
그에비해 sans serif는 이러한 디테일이 없습니다. 대신에 글자는 직선적이고 flat edges를 가집니다.
Arial이나 Helvetica가 이러한 부류입니다. 현재 사용중인 글씨체 역시 sans serif입니다.

그 다음으로는 font-size입니다.
이름에서 볼 수 있듯이, 폰트의 사이즈를 설정합니다.

font-size: 30px;


처럼 사용합니다.

다음은 font-weight입니다.
font-weight는 font가 얼마나 굵고 얇은지를 선택합니다. 값으로 줄 수 도 있습니다.
기본은 normal입니다.(400)
bold로 하면 굵어집니다. (700)
100과 같이 더 작은 값을 주면 얇아집니다.

font-weight: 400;


과 같은 방식으로 사용할 수 있습니다.

다음은 font-style>입니다.
italic을 주면 기울어지게 됩니다.

font-style: italic;


과 같이 사용합니다.
기본은 normal입니다.

다음은 word-spacing입니다.
단어 사이의 간격을 조절합니다.
즉, 스페이스바의 크기를 조절한다고 생각하시면 이해하기 편합니다.
0.25em이 디폴트이며 값이 커지면 거리가 커지게 됩니다.
0.5em 입니다.

word-spacing: 0.5em;


위와 같이 사용할 수 있습니다.

다음은 letter-spacing입니다.
이는 단어 단위가 아닌 한 글자 단위의 간격을 조절합니다.
기술적 용어로는 tracking이라고 부릅니다.
0.5em 입니다.

letter-spacing: 0.5em;


마찬가지로 위와 같이 사용합니다.

다음은 text-transform입니다.
텍스트를 전부 대문자 혹은 소문자로 변환합니다.
uppercase면 대문자, lowercase면 소문자입니다.
기본: This is Sparta!
uppercase: This is Sparta!
lowercase: This is Sparta!

text-transform: uppercase;


사용은 위와 같습니다.
주로, 제목을 전부 대문자로 해준다던지 등에서 사용합니다.
본 페이지 역시 목차부분에 text-transform: uppercase;가 사용되어 있습니다.

다음으로 볼 것은 text-align입니다.
이 친구는 텍스트의 정렬을 결정합니다.
left / center / right를 지정해줄 수 있습니다.
가령, center를 사용 시 다음과 같습니다.

text-align: center;


와 같이 사용합니다.

그 다음은 color입니다.
글자의 색을 지정합니다.
색을 지정하는 다양한 방법에 대해서는 하단의 color에서 다루도록 하겠습니다.
일단은, 색 이름으로 색을 지정해주도록 하겠습니다.
가령, 빨간색이나
노란색 등 색상을 지정해줄 수 있습니다.

color: red;
color: yellow;


사용법은 위와 같습니다.

color와 비슷한 친구로 background-color가 있습니다.
이 친구는 배경색을 지정해줍니다.

background-color: green;


사용은 다음과 같이 해줍니다.

다음은 Opacity입니다.
이는 투명도를 이야기합니다.
0에 가까울수록 투명하고,
1에 가까울수록 불투명합니다.

opacity: 0.3;


사용법은 위와 같습니다.

다음으로 line-height에 대해 보도록 하겠습니다.
이 친구는 줄의 높이를 설정하게 해줍니다.

값을 줄 때 단위를 생략하고 2와 같이 주게 되면, font size에 비례해서 동작합니다.
그 외에 단위를 주며 직접적으로 크기를 지정해줄 수 도 있습니다.

line-height: 2;
line-height: 25px;


위는 눈으로 보기 위해 background-color까지 넣어준 모습입니다.
실제 사용은 위의 코드와 같이 하면 됩니다.

이번에는 font-family를 지정해주는 방법 중에서 Fallback Fonts에 대해 보도록 하겠습니다.
이는 지정된 폰트가 컴퓨터에 설치되어 있지 않은 경우, 다음 순번의 폰트를 지정해주는 방법입니다.
가령, h1 { font-family: "Garamond", "Times", serif; }와 같은 syntax가 있다면
이는 Garamond라는 폰트를 우선적으로 사용하고, 이 폰트가 없으면 Times New Roman폰트를 사용하고
이마저도 없으면 아무 설치되어 있는 serif폰트를 사용한다는 의미입니다.
하지만 유저가 모든 폰트를 컴퓨터에 갖고 있기는 힘들기 때문에, Linking Fonts방식을 사용합니다.
이는 링크를 걸어서 폰트를 사용하는 방법입니다.
<head>
   <link href="https://fonts.googleapis.com/css?family=Droid+Serif" type="text/css" rel="stylesheet">
</head>
 와 같이 넣어주게 되면 지정된 폰트를 사용할 수 있습니다.
혹은 위의 Link에서 링크를 거는 방법 대신에 폰트를 가져오는 방법이 있습니다.
@font-face방법입니다.
link를 HTML document에 넣는 것 대신에, 그 링크를 browser의 URL bar에 집어넣습니다.
브라우저는 CSS rule을 읽을 것이고, 우리는 거기서 /*latin*/으로 묶인 부분들을 발견할 수 있습니다.
이 latin 라벨된 CSS rule을 복사하고 style.css의 맨 위에 붙여넣기 합니다.
그러면 link를 걸지 않고도 해당 폰트를 사용할 수 있습니다.
혹은, @font-face를 사용하여서 이미 설치된 font를 사용하게 할 수 도 있습니다.

@font-face{
font-family: "glegoo";
src: url(../fonts/Glegoo-Regular.ttf) format('tryetype'),
     url(../fonts/Glegoo-Bold.ttf) format('tryetype') }


와 같이 내장된 폰트를 사용하게 할 수 있습니다.



4. Box Model

이번에는 Box Model에 대해 살펴보도록 하겠습니다.

Box Model은 위와 같습니다.
margin > border > padding > content순으로 이루어져 있습니다.
그리고 content에는 width와 height가 있습니다.
하나하나씩 봐보도록 하겠습니다.
width는 너비입니다. content의 좌우 크기를 나타냅니다.
height는 높이입니다. content의 상하 크기를 나타냅니다.

border는 element를 감싸는 선입니다.
border는 3가지 요소로 이루어져 있습니다.
width는 선의 굵기를 나타냅니다. pixel단위로 설정해주거나 thin, medium, thick으로 설정이 가능합니다.
style은 border의 디자인입니다. none, dotted, solid등 10개가 있습니다.
color는 색상입니다.
이 세 요소를 순서적으로 배치해주시면 됩니다.

border: 2px dotted yellow;


현재 이 syntax에 적용되어있는 border는 위와 같습니다.
이 border를 각지게 하지 않고 양 네 귀퉁이를 둥글게 만들어주는 방법도 있습니다.
border-radius를 통해서 가능합니다.

border-radius: 7px


위 처럼 pixel로 주거나 혹은 퍼센트로도 줄 수 있습니다.

다음은 padding입니다.
padding은 content와 border사이의 공간입니다.
padding: 4px;과 같이 상하좌우 4방향 한 번에 지정해줄 수 있고,
padding-top, padding-right, padding-bottom, padding-left로 각각 줄 수 있습니다.
padding: 6px 11px 4px 9px;식으로 값을 줄 경우, 자동적으로 top right bottom left순으로 들어가게 되고
padding: 5px 10px;식으로 2개의 값을 준다면 (top bottom) (left, right)로 묶여서 들어가게 됩니다.

마지막으로는 margin이 있습니다.
margin은 border바깥의 공간입니다. 전체 공간의 크기를 결정합니다.
마찬가지로 4방향 전부, 따로 줄 수 있습니다.
margine-top, margine-right등 동일하게 사용합니다.
padding과 동일하게 한 번에 4개의 값을 주거나 2개의 값을 주거나 등등 모두 가능합니다.

이제는 활용을 보도록 하겠습니다.
먼저, margin: 0 auto입니다.
이 syntax는 상하 margin은 0이 되고, 좌우 마진은 알아서 내용물이 가운데에 배치되도록 조정됩니다.
margin을 주다보면 상하좌우에 있는 다른 요소와 collapse가 발생할 수 있습니다.

그럴 때는 위와 같이, 좌우 마진은 중첩되고 상하 마진은 더 큰 쪽만 반영됩니다.

width와 height는 브라우저 환경에 따라 변동이 되기도 합니다.
그럴때를 대비해서, 최소와 최대값을 미리 지정해줄 수 있습니다.
min-width, max-width, min-height, max-height를 통해서 지정해줄 수 있습니다.
이를 지정해주면, 스크린이 커지거나 작아져서 문제가 생기더라도 해당 영역 안에서만 움직이게 됩니다.
그러다보면 안에 있는 내용물이 넘칠 수 밖에 없는 상황이 생길 수 있습니다.
이럴때는 overflow를 통해서 컨트롤해줄 수 있습니다.
이는 content가 parent를 넘어서 커지는걸 컨트롤해주게 됩니다.
hidden은 넘쳐버린 content를 숨겨버립니다. scroll은 스크롤바를 생성하고 visible은 그냥 삐져나간채로 보입니다. visible이 default입니다.
overflow: visible;식으로 지정을 해주면 됩니다.

넘치는걸 별개로, 그냥 view에서 숨기거나 보여주게 설정할 수 있습니다.
visibility를 사용하면 가능합니다.
hidden은 숨기고, visible은 보여줍니다.
visibility: hidden;식으로 사용해주시면 됩니다.

박스의 크기를 어떤 것을 기준으로 계산할 것인지에 대해서도 알아야 합니다.

box-sizing은 이를 조절하게 해줍니다.
content-box는 콘텐트 영역을 기준으로 정합니다. 기본값입니다.
border-box는 테두리를 기준으로 크기를 정합니다.
initial은 기본값으로 설정합니다.
inherit은 부모 요소의 속성값을 상속합니다.
box-sizing: content-box;식으로 사용합니다.

그 다음으로는 block(box)의 위치를 잡는 방법에 대해서 보도록 하겠습니다.
position은 이를 조정하게 해줍니다.
static(default), relative, absolute, fixed의 4가지 중 하나의 값을 가집니다.
static의 경우는 가장 기본적인 위치입니다.
현재 아무것들도 만지지 않은 지금의 그 모습 자체라고 생각할 수 있습니다.
relative는 이 static에 대해 상대적인 위치를 지정합니다.
가령,

이렇게 될 수 있습니다.


이 코드는 현재

position: relative;
top: 50px;
right: 150px;


의 값을 준 상태입니다.
top은 개체를 아래로 보내고, bottom은 위로, left는 오른쪽으로, right는 왼쪽으로 보냅니다.
다음으로 absolute에 대해 볼 수 있습니다.
이는 view의 좌상단을 기준으로 top, left, right, bottom만큼 움직이게 됩니다.

absoulte


실제 적용된 모습입니다. 목차 좌측의 absoulte가 적용된 것입니다.
적용된 syntax는

position:absolute;
top:50px;
left:50px;


위와 같습니다.
마지막으로, fixed가 있습니다.
이 친구는 조금 독특한데, 화면을 스크롤하던 어쩌든간에 위치가 그냥 view에서 고정됩니다.

다음으로는 z-index가 있습니다.
위의 position을 건드리며 영역들이 겹치기도 하는 모습을 보셨을 겁니다.
이 때 어느것이 앞으로 나오고 어느것이 뒤로 가게 할 지 직접 지정해주는 값이 z-index입니다.
z-index값이 클 수록 개체는 앞으로 나오게 됩니다.

display에 대해 보도록 하겠습니다.
한 content가 있을 때 이 content가 browser의 수평적 공간을 모두 차지하지 않고 있다면, 옆에 나란히 다른 element가 올 수 있을지를 정합니다.
값으로는 inline, block, inline-block이 있습니다.
inline은 tight하게 딱 필요한 공간만 사용합니다. width나 height를 줄 수 없고 필요하지도 않습니다. (딱 content크기에 알맞게 되기 때문)
<em>, <strong>, <a>같은 애들은 default로 inline을 가집니다.
block은 좌우를 꽉 채워서 사용합니다.
width를 주어서 조절이 가능하고, height역시 부여 가능합니다.
<h1> ~ <h6>이나 <p>, <div>, <footer>같은 애들은 default로 block입니다.
inline-block>은 각 특정한 차원을 가지고 같은 줄에 배치됩니다.
width와 height를 주어야 합니다만 크기가 알아서 잘 조절됩니다.

다음으로는 float입니다.
float은 '가능한한' 어느 방향으로 보낼 것인지를 결정합니다.
float:left;나 float:right;식으로 사용합니다.

마지막으로, clear입니다.
float는 여러 요소를 한 번에 띄우는데도 사용이 되지만, 이 요소들의 height가 각각 다르다면 페이지에서의 그들의 레이아웃에 영향을 끼칠 수 있습니다.
구체적으로 말하면 요소들이 서로 충돌할 수 있습니다. 그리고 다른 요소들이 적합하게 왼쪽이나 오른쪽으로 이동하는 것을 허락하지 않을 수 있습니다.
clear는 요소들이 페이지에서 서로 충돌할 때 어떻게 행동하게 할 지를 정할 수 있습니다.
clear:left - 요소의 왼쪽 측면은 어떤 다른 요소들도 오지 않는다.
clear:right - 요소의 오른쪽 측면은 어떤 다른 요소들도 오지 않는다.
clear:both - 요소의 양 쪽 측면 모두 어떤 다른 요소들도 오지 않는다.
clear:none - 요소의 양 쪽 측면 모두 다른 요소들이 올 수 있다.
로 사용이 가능합니다.



5. Color

다음으로는 color에 관하여 보도록 하겠습니다.
색상에는 두 종류가 있습니다.
forground color는 그 요소가 보이는 색상입니다. css에서는 color로 사용합니다.
background color는 그 요소의 배경 색상입니다. css에서는 background-color로 사용합니다.
이 색상을 나타내는 방법을 보겠습니다.
먼저 가장 간단하게는, 색상의 지정된 이름이 존재시 그 이름을 사용할 수 있습니다.
가령, color: coral;이나
color: antiquewhite;등 이름으로 사용할 수 있습니다.

다음으로는 hexadecimal방법이 있습니다.
색깔을 이름으로 쓰는 것이 아니라 RGB로 쓰는 것인데, # R G B 순서로 값을 주게 됩니다.
특이한 점은, RGB를 각각 숫자로 주는 것이 아니라 16진수 두 자리로 주게 됩니다.
색 이름을 쓰는 것은 혹시라도 변경될 우려가 있지만, hexa로 쓰는 것은 변할 일이 (아직은) 없습니다.
방식은 color: #ffaabb;식입니다.
위는 R:ff G:aa B:bb로 RGB값을 준 것입니다.

다음은 우리가 가장 익숙한 RGB colors입니다.
color: rgb(255, 170, 187);식으로 주면 됩니다.

다른 방법으로는 HSL방식이 있습니다.
Hue(색도)는 0부터 360까지의 숫자(각도)입니다. 0은 RED, 120은 GREEN, 240은 BLUE입니다.
Saturation(채도)는 얼마나 강하고 연한지를 나타냅니다. 100%에 가까워질수록 강렬해지고, 0%에 가까워질수록 연해집니다(회색이 됩니다).
Lightness(명도)는 얼마나 밝은지 어두운지를 나타냅니다. default는 50%입니다. 높아지면 밝아지고 낮아지면 어두워집니다.
경우에 따라, HSL이 아니라 HSV로 읽고 쓰기도 합니다.
사용 예시로는, color: hsl(0, 50%, 50%);와 같이 사용합니다.

마지막으로, Opacity와 Alpha에 대해 보도록 하겠습니다.
우리가 보는 대부분의 색상은 불투명합니다. 두 개의 불투명한 요소를 겹쳐서 보면 아래의 요소는 위의 요소에 가려져서 아예 보이지 않습니다.
이에 우리는 투명도(Opacity)를 주어서 뒤가 비치게 할 수 있습니다.
가령, 지금 이 text가 나가는 뒤의 검정 배경의 경우 뒤의 그림이 비쳐서 보이고 있습니다.
이런 식으로 뒤가 비치게 하는 것을 투명도를 통해 조정할 수 있습니다.
hsl대신에 hsla를 사용 시 마지막 매개변수로 투명도를 줄 수 있습니다.
이 마지막 매개변수를 alpha라고 하며 0에서 1까지 값을 줄 수 있습니다.
0에 가까울수록 투명하고, 1에 가까울수록 불투명합니다.
rgb역시 rgba를 사용함으로써 alpha를 줄 수 있습니다.
가령, 현재 뒤의 배경의 색상은 rgba(0, 0, 0, 0.85)입니다.
